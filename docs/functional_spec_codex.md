# 기능 명세서 (Codex용)

> 목적: Codex 또는 AI 개발 에이전트가 바로 구현을 시작할 수 있도록
> 기능을 세분화한 명세서

## 공통 규칙

-   모든 AI 생성 결과는 **근거(Evidence)** 와 함께 반환한다.
-   AI는 사용자가 제공하지 않은 경력, 성과, 수치를 생성하면 안 된다.
-   모든 프로젝트는 Project ID를 기준으로 관리한다.
-   하나의 프로젝트는 여러 출처(GitHub, Notion, Blog 등)를 가질 수 있다.

------------------------------------------------------------------------

## F-01 사용자 인증

### 목적

사용자를 식별하고 모든 포트폴리오와 분석 결과를 계정 단위로 관리한다.

### 설명 및 사용자 시나리오

사용자가 회원가입 후 로그인하면 개인 작업 공간이 생성된다. 이후 등록한
GitHub, Notion, PDF, 분석 결과는 모두 계정에 저장된다.

### 입력

-   이메일 또는 OAuth(GitHub, Google)

### 출력

-   Access Token
-   사용자 프로필

### 완료 조건

-   로그인 성공
-   JWT 발급
-   사용자 정보 조회 가능

------------------------------------------------------------------------

## F-02 채용공고 등록

### 목적

분석 대상 채용공고를 시스템에 등록한다.

### 설명 및 사용자 시나리오

사용자는 URL을 붙여넣거나 이미지를 업로드한다. 백엔드는 본문을 추출하고
하나의 Job Posting으로 저장한다.

### 입력

-   URL
-   이미지(PNG/JPG)
-   PDF(선택)

### 출력

-   JobPosting ID
-   추출된 본문

### 예외

-   URL 접근 실패
-   OCR 실패
-   지원하지 않는 형식

------------------------------------------------------------------------

## F-03 채용공고 구조화

### 목적

비정형 공고를 구조화된 데이터로 변환한다.

### 설명 및 사용자 시나리오

LLM은 공고를 분석하여 직군, 필수 기술, 우대 기술, 요구 역량을 JSON으로
변환한다.

### 출력 예시

``` json
{
  "role":"Backend",
  "requiredSkills":["Spring","MySQL"],
  "preferredSkills":["Redis","Docker"],
  "competencies":["문제해결","협업"]
}
```

------------------------------------------------------------------------

## F-04 포트폴리오 등록

### 목적

사용자의 모든 포트폴리오 출처를 등록한다.

### 지원 플랫폼

-   GitHub
-   Notion
-   Velog
-   Tistory
-   개인 홈페이지
-   PDF
-   LinkedIn
-   Google Drive

### 설명 및 사용자 시나리오

사용자는 URL 또는 파일만 입력한다. 시스템은 이후 자동으로 데이터를
수집한다.

------------------------------------------------------------------------

## F-05 포트폴리오 수집

### 목적

각 플랫폼에서 프로젝트 정보를 수집한다.

### 설명 및 사용자 시나리오

GitHub에서는 Repository와 README를, Notion에서는 프로젝트 페이지를,
블로그에서는 프로젝트 관련 글을 추출한다.

### 출력

프로젝트 후보 목록

------------------------------------------------------------------------

## F-06 프로젝트 자동 분리

### 목적

수집된 데이터를 프로젝트 단위로 분리한다.

### 설명 및 사용자 시나리오

예)

GitHub - Chat Server - Shopping Mall

Notion - 쇼핑몰 프로젝트

Velog - 쇼핑몰 트러블슈팅

↓

AI가 동일 프로젝트임을 판단하여 병합 후보를 생성한다.

### 출력

Project 리스트

------------------------------------------------------------------------

## F-07 프로젝트 편집

### 목적

사용자가 AI가 추출한 프로젝트를 수정한다.

### 설명 및 사용자 시나리오

사용자는

-   제목
-   설명
-   담당 역할
-   기술
-   성과

를 수정하거나 삭제할 수 있다.

------------------------------------------------------------------------

## F-08 프로젝트 선택

### 목적

지원서 작성에 사용할 프로젝트를 선택한다.

### 설명 및 사용자 시나리오

AI가 추천한 프로젝트 중

체크박스로

사용 여부를 결정한다.

------------------------------------------------------------------------

## F-09 적합도 분석

### 목적

채용공고와 프로젝트를 비교한다.

### 설명 및 사용자 시나리오

각 프로젝트마다

-   기술
-   역할
-   문제 해결
-   성과

를 비교하여 적합도를 계산한다.

### 출력

-   적합도 점수
-   추천 이유
-   부족한 점

------------------------------------------------------------------------

## F-10 지원서 내용 추천

### 목적

이력서에 들어갈 내용을 생성한다.

### 설명 및 사용자 시나리오

AI는 선택된 프로젝트를 기반으로

-   어떤 경험을 강조할지
-   어떤 기술을 언급할지
-   어떤 프로젝트를 제외할지

를 추천한다.

### 생성 규칙

-   사실만 사용
-   과장 금지
-   출처 연결

------------------------------------------------------------------------

## F-11 추천 근거 제공

### 목적

모든 추천의 이유를 제공한다.

### 설명 및 사용자 시나리오

사용자가

"왜 이 프로젝트를 추천했나요?"

를 누르면

채용공고 문장과 프로젝트 근거를 보여준다.

### 출력

-   채용공고 근거
-   GitHub 근거
-   Notion 근거
-   Blog 근거

------------------------------------------------------------------------

## F-12 부족 역량 분석

### 목적

채용공고 대비 부족한 경험을 찾는다.

### 출력

-   부족 기술
-   부족 경험
-   부족 역량

------------------------------------------------------------------------

## F-13 프로젝트 추천

### 목적

부족한 역량을 보완할 프로젝트를 추천한다.

### 설명 및 사용자 시나리오

예)

부족

-   Docker
-   AWS

↓

추천

"Spring + Docker + AWS 기반 게시판"

포함해야 할 기능까지 제안한다.

------------------------------------------------------------------------

## F-14 분석 결과 저장

### 목적

모든 분석 결과를 저장한다.

### 저장 항목

-   채용공고
-   선택 프로젝트
-   추천 문장
-   적합도
-   분석 시각

------------------------------------------------------------------------

## F-15 분석 이력 관리

### 목적

과거 분석 결과를 비교한다.

### 사용자 시나리오

삼성 지원 결과

↓

네이버 지원 결과

↓

토스 지원 결과

를 비교한다.

------------------------------------------------------------------------

## F-16 AI 생성 안전 규칙

### 목적

LLM Hallucination 방지

### 규칙

-   존재하지 않는 프로젝트 생성 금지
-   존재하지 않는 기술 생성 금지
-   성과 수치 생성 금지
-   출처 없는 문장 생성 금지
-   사용자가 입력하지 않은 내용 작성 금지

------------------------------------------------------------------------

## F-17 시스템 예외 처리

### 처리 대상

-   GitHub 비공개 저장소
-   Notion 접근 실패
-   URL 오류
-   OCR 실패
-   LLM Timeout
-   API Rate Limit

각 오류는 사용자에게 이해하기 쉬운 메시지와 재시도 방법을 제공한다.
