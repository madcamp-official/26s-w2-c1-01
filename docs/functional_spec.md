# 기능 명세서


이 문서는 현재 구현된 채용공고 맞춤 이력서 추천 서비스의 기능 범위와 동작 기준을 정리한다. 세부 API 형식은 `docs/api_spec.md`, DB 구조는 `docs/db_schema_erd.md`를 기준으로 한다.

## 1. 서비스 개요

사용자는 GitHub로 로그인한 뒤 본인의 GitHub 프로젝트와 CV PDF를 등록한다. 이후 채용공고를 URL, 텍스트, 이미지로 입력하면 시스템은 공고 요구사항을 구조화하고, 사용자의 프로젝트/CV와 비교해 추천 프로젝트, 매칭 근거, CV 적합도, 이력서 초안, 부족 역량 보완 프로젝트를 제공한다.

## 2. 공통 원칙

- 모든 사용자 데이터는 로그인 사용자 단위로 분리한다.
- AI가 생성하는 추천, 분석, 이력서 문장은 가능한 한 원문 근거와 연결한다.
- 사용자가 제공하지 않은 경력, 성과, 수치, 기술 사용 경험을 사실처럼 생성하지 않는다.
- URL/이미지/LLM 처리에 실패하면 사용자가 직접 텍스트를 입력하거나 다시 시도할 수 있는 상태를 제공한다.
- GitHub와 CV에서 수집된 프로젝트는 사용자가 편집할 수 있는 프로젝트 단위로 관리한다.

## F-01 사용자 인증

### 목적

사용자를 식별하고 GitHub 수집 권한과 사용자별 데이터 소유권을 관리한다.

### 기능

- GitHub OAuth 로그인 URL을 제공한다.
- GitHub callback에서 사용자 정보를 가져오고 서비스 access token을 발급한다.
- GitHub access token은 암호화해서 저장한다.
- 로그인 사용자의 기본 프로필과 GitHub username을 조회한다.

### 입력

- GitHub OAuth code

### 출력

- 서비스 access token
- 사용자 id, GitHub id, 이름, avatar URL

### 완료 조건

- 사용자가 로그인 후 보호 API를 호출할 수 있다.
- `/me`에서 현재 사용자 정보를 조회할 수 있다.

## F-02 GitHub 프로젝트 수집

### 목적

사용자의 GitHub 저장소와 README를 수집해 프로젝트 후보와 근거를 생성한다.

### 기능

- 사용자의 분석 동의를 받은 뒤 GitHub 저장소 목록을 조회한다.
- 저장소별 README와 GitHub 메타데이터를 수집한다.
- 저장소를 프로젝트로 저장하고, README를 `source_documents`와 `evidences`에 저장한다.
- 프로젝트 설명, 역할, 기술, 성과를 LLM으로 보강할 수 있다.
- 프로젝트 요약 임베딩을 생성할 수 있다.
- 자동 수집되지 않는 저장소는 `owner/repo` 형식으로 수동 추가할 수 있다.

### 입력

- `agreedToAnalyze: true`
- 수동 추가 시 `fullName: "owner/repo"`

### 출력

- 수집 작업 상태
- 프로젝트 목록
- 프로젝트별 근거 id

### 예외

- 분석 동의 누락
- GitHub token 만료 또는 복호화 실패
- GitHub API 조회 실패
- 저장소 full name 형식 오류

## F-03 CV 업로드 및 섹션 관리

### 목적

사용자의 PDF CV를 분석 가능한 섹션과 프로젝트 근거로 변환한다.

### 기능

- PDF 파일을 업로드한다.
- PDF 텍스트를 추출한다.
- LLM 또는 폴백 파서로 CV를 섹션으로 분리한다.
- 같은 파일명으로 다시 업로드하면 기존 CV와 파생 프로젝트/근거를 교체한다.
- CV 섹션 제목과 본문을 수정할 수 있다.
- CV 섹션 수정 시 CV 기반 프로젝트와 근거를 다시 동기화한다.
- CV를 삭제하면 업로드 파일, 섹션, CV 기반 프로젝트, CV 근거를 함께 삭제한다.

### 입력

- PDF 파일
- 섹션 수정 값: `title`, `content`

### 출력

- CV 문서 id
- 파일명, 상태
- 섹션 목록

### 완료 조건

- CV 프로젝트성 섹션이 추천 대상 프로젝트로 반영된다.
- CV 본문은 분석 시 CV 적합도 계산에 사용할 수 있다.

## F-04 프로젝트 목록 조회 및 편집

### 목적

수집된 GitHub/CV 기반 프로젝트를 사용자가 확인하고 이력서 추천에 맞게 정리한다.

### 기능

- 현재 사용자의 활성 프로젝트 목록을 조회한다.
- 프로젝트 제목, 설명, 역할, 기술, 성과를 수정한다.
- 프로젝트별 근거 id를 함께 제공한다.
- `is_archived`가 true인 프로젝트는 기본 목록에서 제외한다.

### 입력

- `title`
- `description`
- `role`
- `skills`
- `achievements`

### 출력

- 프로젝트 상세 정보
- 수정 시각

## F-05 채용공고 등록

### 목적

분석 대상 채용공고를 원문과 구조화된 요구사항으로 저장한다.

### 기능

- URL, 직접 입력 텍스트, 이미지 입력을 지원한다.
- URL 입력 시 HTML에서 visible text를 추출한다.
- URL 페이지 본문이 부족하면 이미지 후보를 찾아 비전 분석을 시도한다.
- 이미지 입력은 base64 data URL 형식을 지원하며 최대 6개까지 처리한다.
- LLM으로 직무, 필수 기술, 우대 기술, 역량을 구조화한다.
- LLM 실패 시 규칙 기반 폴백 구조화를 사용한다.
- 공고 요약과 임베딩을 생성할 수 있다.

### 입력

- `inputType`: `url`, `text`, `image`
- `content`: 문자열 또는 이미지 data URL 배열

### 출력

- 채용공고 id
- 입력 타입
- 추출된 원문
- 상태

### 예외

- URL 접근 실패
- URL에서 충분한 공고 요구사항을 추출하지 못함
- 이미지 형식/크기 오류
- 이미지 OCR/비전 분석 실패
- 빈 텍스트 입력

## F-06 채용공고-프로젝트 분석

### 목적

채용공고 요구사항과 사용자의 프로젝트/CV를 비교해 적합한 프로젝트와 부족 역량을 찾는다.

### 기능

- 채용공고별 분석 작업을 생성한다.
- 프로젝트의 기술, 설명, 성과, 요약 임베딩을 사용해 추천 점수를 계산한다.
- 추천 프로젝트별 순위, 점수, 추천 이유, 매칭 기술, 부족 기술을 저장한다.
- 추천 매칭 근거를 생성하고 `evidences`와 연결한다.
- 사용자의 CV가 있으면 공고와 CV 전체의 적합도 점수를 계산한다.
- CV 적합도에는 매칭 기술, 부족 기술, 섹션 근거, 규칙 기반 점수, 벡터 점수가 포함될 수 있다.

### 입력

- 채용공고 id
- 추천 개수

### 출력

- 분석 작업 상태
- 구조화된 채용공고
- 추천 프로젝트 목록
- 추천별 매칭 근거
- CV 적합도

### 완료 조건

- 완료된 분석 작업은 결과 id를 가진다.
- 추천 프로젝트는 사용자가 소유한 활성 프로젝트만 포함한다.

## F-07 추천 근거 조회

### 목적

추천 프로젝트와 이력서 문장의 출처 근거를 사용자가 확인할 수 있게 한다.

### 기능

- 근거 id로 원문 근거를 조회한다.
- 근거는 프로젝트 또는 채용공고를 통해 현재 사용자 소유인지 확인한다.
- GitHub README, CV 섹션, 추천 매칭 설명 등 다양한 근거 타입을 지원한다.

### 출력

- 근거 id
- 출처 타입
- 출처 URL
- 제목
- 본문
- 연결 프로젝트 id

## F-08 이력서 초안 생성

### 목적

선택한 프로젝트를 기반으로 채용공고에 맞춘 이력서 초안을 생성한다.

### 기능

- 사용자가 선택한 프로젝트 목록과 채용공고 id로 이력서 생성 작업을 시작한다.
- 같은 채용공고와 같은 프로젝트 순서로 이미 생성된 결과가 있으면 기존 결과를 재사용할 수 있다.
- LLM으로 요약, 기술, 프로젝트 경험 섹션을 생성한다.
- LLM 실패 시 프로젝트 설명과 성과를 사용한 폴백 섹션을 생성한다.
- 섹션마다 연결 가능한 근거 id를 붙인다.
- 선택 프로젝트에서 확인되지 않는 요구 기술을 `missingSkills`로 기록한다.
- 부족 기술을 보완할 프로젝트 제안을 생성한다.

### 입력

- 채용공고 id
- 선택 프로젝트 id 목록

### 출력

- 이력서 작업 상태
- 이력서 결과 id
- 제목, 요약
- 섹션 목록
- 부족 기술
- 보완 프로젝트 제안
- 경고 목록

### 예외

- 프로젝트가 하나도 선택되지 않음
- 채용공고를 찾을 수 없음
- 선택 프로젝트를 찾을 수 없음
- 생성 작업 실패

## F-09 부족 역량 및 보완 프로젝트 제안

### 목적

채용공고 요구사항 중 현재 프로젝트/CV에서 충분히 확인되지 않는 역량을 보완할 방법을 제안한다.

### 기능

- 분석 결과와 이력서 생성 결과에서 부족 기술을 계산한다.
- 부족 기술별로 보완 프로젝트 제목, 설명, 목표 기술, 추천 이유를 제공한다.
- 제안은 사용자의 기존 경험을 과장하지 않고 앞으로 만들 수 있는 프로젝트 형태로 작성한다.

### 출력

- 프로젝트 제목
- 프로젝트 설명
- 목표 기술
- 예상 기간
- 추천 이유

## F-10 분석 결과 저장 및 재조회

### 목적

사용자가 분석/생성 결과를 다시 확인할 수 있게 한다.

### 기능

- 채용공고 분석 결과를 `analysis_results`와 관련 테이블에 저장한다.
- 이력서 생성 결과를 `resume_results`와 관련 테이블에 저장한다.
- 작업 상태는 `async_jobs`에서 조회한다.
- 이력서 생성은 동일 입력에 대해 기존 결과를 재사용할 수 있다.

### 저장 대상

- 채용공고 원문과 구조화 결과
- 추천 프로젝트와 점수/이유
- 추천 근거
- CV 적합도
- 이력서 초안
- 이력서 섹션별 근거
- 보완 프로젝트 제안

## F-11 안전 규칙

### 목적

AI 생성 결과의 과장과 환각을 줄인다.

### 규칙

- 존재하지 않는 프로젝트를 생성하지 않는다.
- 확인되지 않은 기술 경험을 보유 경험처럼 쓰지 않는다.
- 확인되지 않은 수치 성과를 생성하지 않는다.
- 근거 없는 추천 사유를 만들지 않는다.
- 원문에서 확인할 수 없는 내용은 경고 또는 부족 역량으로 분리한다.
- 추천 문장에는 가능한 한 근거 id를 연결한다.

## F-12 예외 처리

### 대상

- 인증 실패
- GitHub token 만료
- GitHub repository 접근 실패
- URL 본문 추출 실패
- 이미지 형식 오류
- 이미지 OCR/비전 분석 실패
- PDF 텍스트 추출 실패
- LLM 응답 실패 또는 파싱 실패
- 임베딩 생성 실패
- 존재하지 않거나 다른 사용자의 리소스 접근

### 원칙

- 사용자가 이해할 수 있는 메시지를 제공한다.
- 재시도 또는 대체 입력이 가능한 경우 그 흐름을 막지 않는다.
- 외부 API나 LLM 실패가 전체 사용자 데이터를 손상시키지 않게 한다.

## MVP 범위

현재 구현 기준 MVP 필수 기능:

- GitHub OAuth 로그인
- GitHub 저장소 자동 수집
- GitHub 저장소 수동 추가
- CV PDF 업로드/섹션 편집/삭제
- 프로젝트 목록 조회 및 편집
- 채용공고 URL/텍스트/이미지 등록
- 채용공고 구조화
- 채용공고-프로젝트 추천 분석
- CV 적합도 분석
- 추천 근거 조회
- 이력서 초안 생성
- 부족 역량 및 보완 프로젝트 제안

현재 직접 구현되어 있지 않거나 제한적인 기능:

- Notion, Velog, Tistory, LinkedIn, Google Drive 직접 수집
- 채용공고 PDF 첨부 API
- 결과 PDF/Markdown 내보내기
- 분석 히스토리 비교 전용 화면
